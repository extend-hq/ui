Annotations provide review markup without coupling the app to one file renderer.

## Useful behaviors

- Anchors by page, bounding box, text range, cell range, or image region.
- Highlight, rectangle, pin, note, and comment modes.
- Selection handles, keyboard escape, and collision-aware popovers.
- Read-only and editable modes for review workflows.

## Common slots

| Slot | Purpose |
| --- | --- |
| `annotation` | Rendered mark, pin, or region. |
| `editor` | Comment input, label, color, and status controls. |
| `selection` | Active region and resize handles. |
