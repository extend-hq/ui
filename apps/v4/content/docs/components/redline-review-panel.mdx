Redlines are for document approval flows where users need to inspect changes in context.

## Useful behaviors

- Group changes by author, section, page, date, or status.
- Accept, reject, resolve, and jump-to-change actions.
- Inline summaries for inserts, deletes, moves, and formatting changes.
- Read-only audit mode for finalized documents.

## Common slots

| Slot | Purpose |
| --- | --- |
| `changeItem` | A single proposed edit or comment thread. |
| `actions` | Accept, reject, resolve, and jump controls. |
| `filters` | Author, change type, and status filters. |
